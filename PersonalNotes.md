# 🗝️ Техническое задание: «Сервис личных заметок с сессионной аутентификацией»

## 🧭 Описание проекта
Создать веб-сервис на **FastAPI**, где пользователи могут вести **личные заметки**.  
Каждый пользователь имеет собственный кабинет, в котором он может просматривать, добавлять, изменять и удалять **только свои заметки**.  
Доступ к функционалу обеспечивается через **аутентификацию по сессии (session cookies)**.

---

## 🎯 Цель проекта
Научиться работать с **сессионной аутентификацией** в FastAPI:
- сохранять состояние пользователя между запросами;
- использовать cookies для хранения сессионного идентификатора;
- реализовать разграничение доступа (авторизацию).

---

## 👥 Роли системы
### 1. Пользователь (User)
- Может зарегистрироваться, войти, выйти из системы.
- После входа получает cookie-сессию.
- Может работать только со своими заметками.

### 2. Администратор (Admin)
- Имеет отдельный логин.
- Может просматривать заметки всех пользователей.
- Может блокировать пользователей или удалять их заметки.

---

## ⚙️ Основные функции

### 🔐 Аутентификация
1. **Регистрация** — создание нового пользователя (email, username, password).  
2. **Вход (Login)**  
   - После проверки логина и пароля сервер создаёт **сессию** и отправляет пользователю cookie.  
   - Cookie используется для последующих запросов.  
3. **Выход (Logout)**  
   - Удаление cookie и сессии с сервера.  

---

### 🗒️ Работа с заметками
1. Создание новой заметки (title, content, дата).  
2. Просмотр всех своих заметок.  
3. Просмотр одной заметки по ID.  
4. Редактирование и удаление заметки (только своей).  
5. (Для админа) — просмотр всех заметок всех пользователей.  

---

### 🚫 Авторизация
- Пользователь без активной сессии не имеет доступа к API заметок.  
- Попытка доступа без cookie возвращает ошибку **401 Unauthorized**.  
- Один пользователь не может читать или менять чужие заметки.  

---

## 📦 Технические требования
- **Framework:** FastAPI  
- **Аутентификация:** Session-based (через cookies)  
- **Управление сессиями:** in-memory (dict), Redis, файл или база данных  
- **База данных:** SQLite, PostgreSQL или другая  
- **Формат ответов:** JSON  
- **Dependency (`Depends`)** используется для проверки активной сессии  

---

## 🌟 Дополнительные возможности (по желанию)
- Опция “запомнить меня” — долговечная cookie.  
- Категории заметок (личная, работа, учёба).  
- Страница профиля пользователя с количеством заметок.  

---

## 🔗 Возможные эндпоинты API (примерная структура)

| Метод | Эндпоинт | Описание |
|-------|-----------|----------|
| `POST` | `/register` | Регистрация нового пользователя |
| `POST` | `/login` | Вход в систему, создание сессии |
| `POST` | `/logout` | Выход, удаление cookie |
| `GET` | `/notes` | Просмотр всех заметок пользователя |
| `POST` | `/notes` | Создание новой заметки |
| `GET` | `/notes/{note_id}` | Просмотр конкретной заметки |
| `PUT` | `/notes/{note_id}` | Изменение заметки |
| `DELETE` | `/notes/{note_id}` | Удаление заметки |
| `GET` | `/admin/notes` | (Админ) просмотр всех заметок |
